📚 ナレッジ化・ドキュメント整備フロー

「毎回同じ説明をしなくていい」環境を作るClaude Coworkの使い方

ナレッジ管理の仕組み

📋
CLAUDE.md（AIへの指示書）
各プロジェクトフォルダに配置するAIへの「説明書」。クライアント名の正式表記・禁止事項・作業ルールを書いておくと毎回の説明がゼロになる。
例：「このプロジェクトでは『マツキヨコカラ』は誤表記。必ず『マツキヨコカラ』を使う」→ 以後チェック不要
🧠
メモリシステム
AIが自分の好み・フィードバック・プロジェクト情報を長期記憶するファイルを自動管理。セッションをまたいでもコンテキストを引き継げる。
例：「前回の修正フィードバック」「よく使う表現の好み」「プロジェクトの背景」を自動保存
🔖
チェックポイント・引き継ぎ
/checkpoint コマンドで作業状態をスナップショット保存。翌日や別PCで「○○の続き」と言うだけで再開できる。
例：「今日はここまで」→「明日の続き」でも前回の文脈がそのまま引き継がれる
⚡
Skill化（パターンの再利用）
「一度やった作業フロー」をスラッシュコマンドとして保存。/ssd-sync・/gas-push・/reply-assist など何でも1コマンドに。
例：「/ssd-sync push」でローカル→SSD→GitHubへの同期が全自動実行
📁
rules フォルダ（ルール集）
表記ルール・コーディングスタイル・安全ルールなどを~/.claude/rules/ に集約。全プロジェクトに自動適用される。
例：「HTMLを作ったら必ずOGPタグを入れる」「ファイルは削除せず退避フォルダへ」
🔄
session-handoff（セッション引き継ぎ）
セッション終了時に「今日やったこと・未完了・次回やること」をCLAUDE.mdに自動追記。次回セッションで状況確認不要。
例：「今日はここまで」と言うだけでSession Handoffが自動生成・保存される

メモリの種類と使い分け

🧠 AIが長期記憶する4タイプ
user
ユーザー情報：自分の役職・得意分野・好み。「Issyさんは日本語ネイティブ・デザイン重視」など、毎回説明しなくて済む情報
feedback
フィードバック：「この表現はNG」「このアプローチは好き」などの修正履歴。同じミスを二度しない
project
プロジェクト情報：案件の背景・制約・締め切り。「MCCMの報告会は〇月〇日」など時系列情報を保存
reference
参照先：「バグはLinear INGEST プロジェクトで管理」「監視はGrafanaの〇〇ダッシュボード」など外部リソースの場所
