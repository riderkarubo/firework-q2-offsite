📊 Case Study — MCCM

年次報告会の資料を
Claude Codeで作るまで

実際の資料・データをAIに読み込ませ、
アジェンダごとに深掘り分析→GASでGoogleスライドへ自動出力するプロセス

📁
Phase 1
あらゆる資料をClaude Codeに格納
プロジェクトフォルダにすべての素材を集め、CLAUDE.md に背景・ルール・禁止事項を記述。
AIがセッションをまたいでも文脈を忘れない「永続記憶」として機能させる。
📊 配信数値まとめ（CSV） 📈 競合他社ライブ配信数値 🎯 QBR・KPI設計資料 📝 台本・分科会資料 📋 過去の振り返り資料 🏢 メーカー実績レポート 🤝 協賛配信のあゆみ ⚙️ CLAUDE.md（AI指示書）
🧠
Phase 2
社内すり合わせ済みの企画骨子を読み込ませる
社内で整理した提案骨子（アジェンダ・発表時間・伝えたいメッセージ）をClaude Codeに渡し、
全体の構成・論点・必要な分析を一緒に整理する。

🔍
Phase 3
アジェンダごとに骨子を深掘り・分析させる
各アジェンダに対して「何を言いたいか → 何のデータが必要か → どう見せるか」をAIと壁打ちし、
必要な分析をその場でClaude Codeに実行させる。
📌 A1 — 資産価値の可視化
累計視聴数の推移グラフ化 前年比+62%の根拠整理 7経営資産の言語化 協賛19社実績まとめ
📌 A2 — KPI設計 & 競合比較
競合比較：なぜ低いか言語化 KPI再設計の壁打ち 競合他社数値CSVを分析 2026目標KPI策定
📌 A3 — エンゲージメント比較
PB回 vs 協賛回 比較 協賛回 vs レギュラー回 フォロリポ分析 コメント内容テキスト分析 お悩み訴求チェック
📌 新メニュー / キャンペーン分析
Xフォロリポ×アーカイブ効果 KATE好事例の分析 キャンペーン有無の洗い出し 月5回受注の方程式
💬 実際のやり取り例：「A2の競合比較、なぜ視聴者数が低いか原因を3つに絞って言語化して、根拠データをCSVから引っ張って」
→ Claude Codeがデータを読み込み、分析結果を即座に構造化されたMarkdownで返す
✍️
Phase 4
統合版ドキュメントを生成・精査
各アジェンダの分析結果・骨子・数値・メッセージを1つのHTMLドキュメント（統合版）に集約。
スライドに落とす前の「完成原稿」として社内でレビュー・フィードバックを行う。
📄
提案に向けた情報まとめ_統合版.html
分析結果・KPI設計・戦略骨子・メーカー実績・競合データを
アジェンダ順に1ファイルにまとめたHTMLドキュメント。
フィードバック→修正→再生成を短サイクルで回す。
🚀
Phase 5
GAS でGoogleスライドに自動出力
Claude Codeが生成したGoogle Apps Script（GAS）コードを実行し、
統合版ドキュメントの内容を構造化されたGoogleスライドに自動変換・整形する。
⚡
GAS 自動スライド生成
アジェンダ構成・データ・グラフ・デザインルールをコードに落とし込み、
ワンクリックでGoogleスライドへ展開。手作業のコピペ・整形をゼロにする。
📝 原稿テキスト → スライド本文に自動配置
📊 数値データ → グラフに自動変換
🎨 レイアウト・フォント → 一括適用
📑 アジェンダ構成 → セクション分割
14+ Claude Codeが生成した分析レポート数
60分→ 従来の手作業分析を壁打ちで大幅短縮

AIが「考える相棒」として
資料制作に参加する

従来の「資料を作る」作業は、
データを探す・数字を拾う・文章を書く・スライドを整える…
すべてが人間の時間を消費していた。

Claude Codeを使うことで、「何を伝えるか」の判断だけに集中し、
それ以外のすべてをAIに任せられる体制ができた。

📂
素材格納 → AIが全データを記憶・参照
🧠
骨子読込 → 構成と論点を即座に把握
🔍
深掘り分析 → データから示唆を自動抽出
✍️
原稿生成 → 統合ドキュメントに自動集約
⚡
GAS出力 → Googleスライドに自動変換
